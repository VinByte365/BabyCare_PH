# BabyGuide PH Tech Stack

This document lists the technologies currently used across the BabyGuide PH project.

## Project Structure
- `frontend/`: Expo managed React Native mobile app written in TypeScript
- `backend/`: FastAPI REST API written in Python
- Root docs: implementation, product, design, and planning references

## Frontend
- Expo SDK 54
- React Native 0.81
- React 19
- TypeScript 5.9
- Expo managed workflow

## Frontend Navigation
- React Navigation
- `@react-navigation/native`
- `@react-navigation/bottom-tabs`
- `@react-navigation/native-stack`
- `react-native-screens`
- `react-native-safe-area-context`
- `react-native-gesture-handler`

## Frontend State and Data
- Zustand for local app state
- TanStack Query for server state and API request caching
- AsyncStorage for persisted non-sensitive local data
- Expo SecureStore for secure token storage
- NetInfo for network/offline state detection

## Frontend Device Features
- Expo Camera for live camera capture
- Expo Image Picker for gallery/photo uploads
- Expo Haptics for tactile feedback
- Expo Linking for phone, maps, and external links
- Expo Splash Screen for launch screen control
- Expo Font with Google Fonts packages
- DateTimePicker for date selection

## Frontend UI and Motion
- Lottie React Native for animations
- React Native Reanimated
- React Native Worklets
- Expo Vector Icons
- Custom theme tokens in `frontend/src/theme/`
- Inter and JetBrains Mono font packages

## Backend
- Python
- FastAPI
- Uvicorn
- SQLAlchemy 2
- Pydantic 2
- Pydantic Settings
- Python Multipart for form/file request support
- HTTPX
- Python Dotenv

## Backend API
- REST API under `/api/v1`
- FastAPI routers and endpoint modules
- OpenAPI/Swagger docs generated by FastAPI
- ReDoc generated by FastAPI

## Database and Migrations
- PostgreSQL by default
- AsyncPG
- Psycopg2 Binary
- SQLAlchemy ORM models
- Alembic dependency is installed, but migrations are not currently committed
- SQLite compatibility exists for local debugging/tests

## Authentication and Security
- JWT authentication with `python-jose`
- Password hashing with Passlib and bcrypt
- OAuth2 password login flow
- Environment-based configuration through `.env`
- Secure mobile token storage through Expo SecureStore

## Offline and Local Capability
- Bundled local disease and symptom library logic
- AsyncStorage-backed offline queues
- NetInfo-backed network state tracking
- Offline banner component for user feedback
- Analytics event queueing for later sync

## Computer Vision Skin Check
- Expo Camera for live image capture
- Expo Image Picker for uploaded skin photos
- Planned image classification classes:
  - Measles
  - Heat Rash
  - Chickenpox
  - Eczema
  - Normal Skin
- Model runtime/inference architecture is still to be finalized: on-device model bundle or backend inference endpoint

## Development Commands
- Frontend start: `npm start`
- Frontend Android: `npm run android`
- Frontend iOS: `npm run ios`
- Frontend web: `npm run web`
- Backend run: `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`

## Testing and Quality Tooling
- No committed frontend test runner is currently configured
- No committed backend test suite is currently configured
- No committed ESLint, Prettier, Ruff, Black, or formatter configuration is currently configured
- TypeScript strict mode is enabled for the frontend
